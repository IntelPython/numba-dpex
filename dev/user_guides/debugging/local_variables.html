Local variables¶

Note

  • NUMBA_OPT=0 “no optimization” level - all local variables of the kernel function are available.

  • NUMBA_OPT=1 or higher - some variables may be optimized out.

Consider Numba-dppy kernel code sum_local_vars.py

15import numpy as np
16import numba_dppy as dppy
17import dpctl
18
19
20@dppy.kernel(debug=True)
21def data_parallel_sum(a, b, c):
22    i = dppy.get_global_id(0)
23    l1 = a[i] + 2.5
24    l2 = b[i] * 0.3
25    c[i] = l1 + l2
26
27
28global_size = 10
29N = global_size
30
31a = np.array(np.random.random(N), dtype=np.float32)
32b = np.array(np.random.random(N), dtype=np.float32)
33c = np.ones_like(a)
34
35with dpctl.device_context("opencl:gpu"):
36    data_parallel_sum[global_size, dppy.DEFAULT_LOCAL_SIZE](a, b, c)
37
38print("Done...")

When GDB debugger hits the last line of the kernel, info locals command returns all the local variables with their values.

Note

Known issues:

  • GDB debugger can show the variable values, but these values may be equal to 0 after the variable is explicitly deleted or the function scope is ended. For more information refer to Numba variable policy.

print variable¶

(gdb) print l1
$3 = 2.5931931659579277
(gdb) print l2
$4 = 0.22954882979393004
(gdb) ptype a
type = byte [56]

Note

Known issues:

  • Kernel variables are shown in intermidiate representation view (with “$” sign). The actual values of the arrays are currently not available.

ptype variable¶

Variable type may be printed by the command ptype variable and whatis variable:

(gdb) whatis a
type = byte [56]
(gdb) ptype l1
type = double
(gdb) whatis l1
type = double
(gdb) continue
...
Done...
