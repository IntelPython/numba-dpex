meminfo_alloc_unchecked(builder, size, usm_type, device)

Allocate a new MemInfo with a data payload of size bytes.

The result of the call is checked and if it is NULL, i.e. allocation failed, then a MemoryError is raised. If the allocation succeeded then a pointer to the MemInfo is returned.

Args:

builder (_type_): LLVM IR builder size (_type_): LLVM uint64 Value specifying the size in bytes for the data payload. usm_type (_type_): An LLVM Constant Value specifying the type of the usm allocator. The constant value should match the values in dpctl's libsyclinterface::DPCTLSyclUSMType enum. device (_type_): An LLVM ArrayType storing a const string for a DPC++ filter selector string.

Returns: A pointer to the MemInfo is returned.
